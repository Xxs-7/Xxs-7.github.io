从 0 到 1 实现一个博客

本文讲解一下实现博客的思路，并不涉及具体的实现细节。

需要做的事情

  1. 调研（UI配色，功能）。
  2. 自己的博客要具备哪些功能。
  3. 编写代码
  4. 确定博客的部署，是部署到个人服务器还是第三方托管平台，使用什么工具进行持续集成。

借鉴

优秀个人独立博客导航 建站于 2006 年，收录了不少优秀的个人独立博客。

博客功能

博客功能核心是内容显示，其次是评论、搜索、点赞、分享、归档、排序、主题。 至于内容显示，还可以结合第三方组件，比如 codesandbox，algolia，...

技术栈

建站框架十分多，如果不关注技术，纯粹想快速建站：

  • WordPress（主流） 是一个以 PHP 和 MySQL 为平台的自由开源的博客软件和内容管理系统。 WordPress 具有插件架构和模板系统。 截至 2018 年 4 月，排名前 1000 万的网站中超过 30.6% 使用 WordPress，是世界上最受欢迎的 CMS。
  • Blogger 是由 Google 提供的免费博客平台，使用简单，可以快速创建和发布博客文章。虽然功能相对简单，但对于只需简单博客的用户来说是一个不错的选择。 如果关注技术，想学习知晓整个技术栈：
  • VitePress 是一款基于 vite vue3 的静态站点生成器
  • NextJs 是一个 React 框架，它允许你使用 React 框架建立超强的、有利于 SEO 的、极度面向用户的静态网站和网络应用。
  • Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他标记语言）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
  • Gatsby is a free and open source framework based on React that helps developers build blazing fast websites and apps

本博客使用的是 NextJs 构建的 SSG 网站：

  • 纯静态，在实现功能的时候考虑的是一个渐进式，功能持续添加。一个软件的 2.0 版本才是相对成熟的。
  • 结合 mdx 技术，使用 markdown 格式编写文档，通过实现自定义组件进行展示。

部署

当博客项目的代码制作完毕后，需要考虑部署在哪里了：

  • 个人服务器：需要购买域名并配置公网IP。曾经买过一次腾讯云的服务器，新人优惠才需要几十块，而后价格过高就没买了，当时在服务器上部署了 gitlab 和 gitlab-runner 做持续集成。但是个人服务器需要进行维护来防御网络攻击，有几次网站 dump 掉了。并且费用过高，没必要这么搞。
  • 第三方托管平台：例如 github，vercel。本博客部署在 github 上，有免费的域名，并且结合 github action 可以方便的进行 code review 和持续部署。缺点就是访问不太稳定。
